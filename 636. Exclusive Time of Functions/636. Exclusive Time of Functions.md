# 636. Exclusive Time of Functions

## Problem Statement

On a single-threaded CPU, we execute a program containing `n` functions. Each function has a unique ID between `0` and `n-1`.

Function calls are stored in a call stack: when a function call starts, its ID is pushed onto the stack, and when a function call ends, its ID is popped off the stack. The function whose ID is at the top of the stack is the current function being executed.

You are given a list `logs`, where `logs[i]` represents the `ith` log message formatted as a string `"{function_id}:{"start" | "end"}:{timestamp}"`.

A function's exclusive time is the sum of execution times for all function calls in the program.

Return the exclusive time of each function in an array, where the value at the `ith` index represents the exclusive time for the function with ID `i`.

## Input
* `n`: An integer representing the number of functions
* `logs`: A list of strings in format "function_id:operation:timestamp"

## Output
* A list of integers where index i represents exclusive time for function i

## Constraints
* `1 <= n <= 100`
* `2 <= logs.length <= 500`
* `0 <= function_id < n`
* `0 <= timestamp <= 10^9`
* No two start events will happen at the same timestamp
* No two end events will happen at the same timestamp
* Each function has an "end" log for each "start" log

## Examples

### Example 1:
```
Input: n = 2, logs = ["0:start:0","1:start:2","1:end:5","0:end:6"]
Output: [3,4]
Explanation:
Time 0-1: Function 0 executes (2 units)
Time 2-5: Function 1 executes (4 units)
Time 6-6: Function 0 executes (1 unit)
Function 0: 2 + 1 = 3 units
Function 1: 4 units
```

### Example 2:
```
Input: n = 1, logs = ["0:start:0","0:start:2","0:end:5","0:start:6","0:end:6","0:end:7"]
Output: [8]
Explanation:
Time 0-1: Function 0 executes (2 units)
Time 2-5: Function 0 (recursive) executes (4 units)
Time 6-6: Function 0 (2nd recursive) executes (1 unit)
Time 7-7: Function 0 executes (1 unit)
Function 0: 2 + 4 + 1 + 1 = 8 units
```

### Example 3:
```
Input: n = 2, logs = ["0:start:0","0:start:2","0:end:5","1:start:6","1:end:6","0:end:7"]
Output: [7,1]
Explanation:
Time 0-1: Function 0 executes (2 units)
Time 2-5: Function 0 (recursive) executes (4 units)
Time 6-6: Function 1 executes (1 unit)
Time 7-7: Function 0 executes (1 unit)
Function 0: 2 + 4 + 1 = 7 units
Function 1: 1 unit
```

## Solution: Stack with Time Tracking Approach

```python
class Solution:
    def exclusiveTime(self, n: int, logs: List[str]) -> List[int]:
        counter = [0] * n
        stack = []
        prev = 0

        for log in logs:
            id, operation, time = log.split(":")
            id, time = int(id), int(time)

            if operation == "start":
                if stack:
                    counter[stack[-1]] += time - prev
                stack.append(id)
                prev = time
                 
            else:  # operation == "end"
                counter[stack[-1]] += time - prev + 1
                stack.pop()
                prev = time + 1
        
        return counter
```

### Complexity Analysis

- **Time Complexity**: O(m), where m is the number of logs
  - Single pass through all log entries
  - Each log processed in O(1) time
  - String split and int conversion are O(1) for bounded input sizes

- **Space Complexity**: O(n), where n is the number of functions
  - Call stack can grow up to n functions (if all nested)
  - Result array of size n
  - Overall: O(n)
